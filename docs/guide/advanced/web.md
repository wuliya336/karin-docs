## 🎯 快速访问

启动 Karin 后，你会在控制台日志中看到类似下面的信息：

```bash
[Karin][00:58:40.722][MARK] karin 启动完成: 耗时 1.20 秒...
[Karin][00:58:40.722][INFO] [server] webui 已启动 访问地址: http://127.0.0.1:7777/web
[server] http鉴权token: AffryQ
[server] ws鉴权token: FGncVt
```

默认情况下，网页控制台运行在:

- 地址：`http://127.0.0.1:7777/web`
- 端口：`7777`
- 鉴权 token: 初始化项目自动生成

## 🔐 登录认证

:::tip 友情提示
第一次登录需要输入 HTTP 认证令牌 (｀・ω・´)
:::

![登录界面](/web/karin_connect.png)

认证令牌就在启动日志中的 `HTTP Auth Token` 字段，复制粘贴就可以啦！

### 修改认证令牌

:::warning 安全提示
如果你怀疑认证令牌已泄露，请立即修改！否则可能导致未经授权的访问 (╥﹏╥)
:::

修改步骤：

1. 找到项目根目录下的 `.env` 文件
2. 修改 `HTTP_AUTH_KEY` 字段的值
3. 重启 Karin 框架 `(其实可以不重启的，Karim 会自动检测到配置文件变化~)`
4. 使用新的认证令牌登录

```bash
# .env 文件示例
HTTP_AUTH_KEY=your_new_secret_key  # 修改这里的值
```

:::tip 友情提示

- 建议使用足够复杂的随机字符串作为认证令牌
- 定期更换认证令牌是个好习惯 (｀・ω・´)
- 请妥善保管认证令牌，不要分享给他人
  :::

Web UI 配置系统新增了 Select 下拉选择组件，让插件配置更加灵活便捷：


## 🤔 常见问题

<div class="faq-section">

### Q: 访问不了控制台怎么办？

确认以下几点：

1. Karin 是否正常启动
2. 端口 `7777` 是否被占用
3. 防火墙是否放行

### Q: 令牌输入正确但登录失败？

可能是因为：

1. 令牌过期
2. 网络连接问题
3. 浏览器缓存问题（试试清除缓存）

### Q: 如何修改默认端口？

在配置文件中`.env` 文件中修改 `HTTP_PORT` 字段即可 (｀・ω・´)
